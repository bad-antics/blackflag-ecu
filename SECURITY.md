# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in BlackFlag ECU:

1. **Do NOT** create a public GitHub issue
2. Email security concerns to the maintainer via GitHub
3. Provide detailed information about the vulnerability
4. Allow 90 days for a fix before public disclosure

## Scope

Security reports should relate to:
- Application vulnerabilities
- Data exposure risks
- Authentication bypasses

Out of scope:
- ECU security research (intended functionality)
- Social engineering
- Physical attacks
